Необхідно реалізувати сервіс управління завданнями, використовуючи можливості
фреймворка laravel.
Додаток повинен складатися з 2-х частин:
1. Адміністративна панель
2. Frontend API
Адміністративна панель
Повинна бути захищена авторизацією та мати інтерфейс з наступними розділами:
1. Список завдань
2. Перегляд/редагування/додавання задачі
3. Видалення завдання
4. Список статусів (готово, в роботі, потрібно зробити)
5. Перегляд/редагування/додавання статусів
6. Видалення статусів
Список полів сутності «Завдання»:
1. Назва – 255 символів
2. Опис – 4000 символів
3. Дата створення - дата/час
4. Дата редагування - дата/час
5. Термін виконання - дата/час
6. Статус - прив'язка до статусу
Список полів сутності «Статус»
1. Назва – 255 символів
2. Сортування – ціле число
Поле статус завдання має бути у вигляді випадаючого списку який заповнюється
зі статусів, створених у розділі «Список статусів».
Frontend API
Повинно реалізовувати такі публічні (доступні без авторизації) методи:
1. Перегляд списку завдань з можливістю фільтрації по всіх полях, крім "опис"
2. Перегляд одного завдання по ідентифікатору
3. Зміна статусу задачі
Розташування та найменування методів, формат та візуалізація на твій розсуд.
Над оформленням візуально можеш не заморочуватись в принципі, так як нам головне сам код. В результаті я маю мати змогу звернутись до твоєї аплікації по АРІ і отримати звідти дані згідно вищеописаних методів.
